详细描述

特殊功能支持

作者
hudx (hudin.nosp@m.gxua.nosp@m.n.xia.nosp@m.n@ha.nosp@m.ier.c.nosp@m.om)

特殊支持1-【8C/8D透传支持】 8C/8D 为E++底板与云端的数据透传功能。本事是直通业务，但出现SmartDevice产品形态后，理论上不应该在支持此业务。 但产业使用的SmartDevice已有应用，所以做兼容支持，违背了设计初衷。 当前8C/8D的应用如下：

  1. 模块应用，透传业务【正确】
  2. SDK应用，设备APP向E++底板发送8C命令，底板回复8D，设备APP收到后，不会转发到云。【异常使用，已有应用】
  3. SDK应用，云端推送8C到设备APP，设备APP将数据解析发送给自有底板（非E++设备），设备APP自行组装8D返回云端。【异常流程，产业测试中】
  4. OTA兼容性接口，当SDR/SDL版本升级至OSDK版本时，需要使用相应接口。

typedef uhsd_s32(* uhsd_dev_epp_transparent_rpt_cb) (uhsd_s32 devHandle, uhsd_s32 src_devHdl, uhsd_char *op_name, uhsd_char *epp_data, uhsd_u32 epp_data_len)

底板设备E++数据回调通知，SDK收到源设备上报数据后触发该回调，开发者实现。

参数
[in]devHandle设备句柄
[in]src_devHdl源设备句柄，0:数据来源为云、小循环（本地、ble）, 非0:数据来源设备为联动设备
[in]op_name= "066D"、"066E"、"067D"、"04"、"06"
[in]epp_dataepp原始数据
[in]epp_data_lenepp原始数据长度
[in]traceId链式跟踪标识
注意
标准E++设备二进制通用接口，支持Wifibase、uwt通信。

◆ uhsd_dev_module_ota_set()

UHSD_API uhsd_s32 uhsd_dev_module_ota_set ( uhsd_moduleHandle  moduleHandle,
uhsd_s32  enable 
)

模组升级设置

扩展接口，设备APP不可见。 为保证此接口不造成额外的使用隐患，目前仅支持4G模组OTA使能设置。

参数
enable1:支持；其他：不支持。
返回
uhsd_s32
返回值
0成功
注意
对于三方模组，设计上，不支持OTA，三方模组OTA需要使用整机升级功能。 为兼容之前的4G模块升级，定义此接口。 设备APP需要使用时，需要技术支持进行告知此接口功能。用户自行extern使用。 正式文档不阐述此接口，切记。

UHSD_API uhsd_s32 uhsd_ota_restore_upgrade ( const uhsd_u8 *  items,
uhsd_u32  items_len 
)

恢复ota升级

扩展接口，设备APP不可见。 此接口只针对SDR/L从老版本直接升级到支持uhomeOS版本后调用 升级参数包含如下： uhsd_u32 upgrade_sn; uhsd_u32 upgrade_flag; uhsd_u32 total_len; uhsd_u8 verification[32]; uhsd_u32 upgrade_dst; uhsd_u8 key[32]; uhsd_u8 url[128];

参数
items升级信息，包含多个项目，为字节数组
items_len升级信息长度,其长度必须为208
返回
uhsd_s32
返回值
0成功
