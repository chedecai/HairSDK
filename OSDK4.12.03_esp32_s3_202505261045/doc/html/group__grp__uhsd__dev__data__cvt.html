)

OP binary数据转标准数据

将E++软件上报的响应（事件/属性）的binary数据模型转换为标准设备数据模型 数据来源： uhepp_op_binary_callback uhsd_dev_device_op_binary_resp uhsd_dev_op_binary_report

参数
[in]dataCvtHandle句柄，来源： uhsd_dev_data_cvt_crt 的返回值
[in]devHandle句柄，来源： uhsd_dev_new_devHandle 的返回值
[in]in_op_name操作类型， 包括1. 全属性查询；2.单属性查询；3. 告警查询。对应 uhsd_dev_op_binary_cb op_name
[in]in_pairs数据对，对应 uhsd_dev_op_binary_cb pairs[]
[in]in_pairs_num数据对个数，对应 uhsd_dev_op_binary_cb pairs_num
[out]out_pairs对应 uhsd_dev_device_op_binary_resp uhsd_dev_op_binary_report
[out]out_pairs_num对应 uhsd_dev_device_op_binary_resp uhsd_dev_op_binary_report pairs_num
[out]out_data_mem_ctx内存释放句柄
返回
uhsd_s32
返回值
0成功
others失败
注意
使用完成out_pairs之后，出参必须使用 uhsd_dev_data_cvt_pairs_free 进行释放

smartDevice 设置需要转换的设备信息

设备APP必须调用此接口成功后，后续的转换接口才可以正常工作。

参数
[in]devHandle数据转换句柄，来源: uhsd_dev_data_cvt_crt
[in]devHandle设备句柄，来源: uhsd_dev_new_devHandle
[in]dc_status_cb回调状态，当转换器状态变更时通过此回调通知调用者。当前状态包括：新数据下载成功(用户可正常使用转换接口)；
[in]user_param用户设置的回调数据指针， 会在 uhsd_dev_data_cvt_status_notify_cb 回调时返回给调用者。
返回
uhsd_s32
返回值
0成功
<0异常 UHSD_ERROR_NUM
无预置设备数据，需要等待网络连接且数据下载成功后，才可使用转换接口。
预制数据异常，需要等待网络连接且数据下载成功后，才可使用转换接口。
